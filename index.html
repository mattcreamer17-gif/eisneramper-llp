The frame

Most firms automating professional services work reach for a language model where a structured context layer and a decision tree would do the job better. Our working 60/30/10 frame applies directly here. Roughly 60% of the recurring review, integration, and workpaper reconciliation work is traditional code and database work. Roughly 30% is rule-based logic that auditors already write in English. 10% is where a language model genuinely earns its place.

Applied to EisnerAmper's post-combination integration work, that is where methodology licensing and interpretability earn partner hours back. Not inside the attest workflow. In the layer above, where firm-wide context lives.

The relevant paper

Interpretable Context Methodology.

Folder Structure as Agent Architecture.

Submitted to ACM TiiS. Open repository on GitHub under MIT license. ICM is the layered context architecture that a professional services firm can license, brand, and ship inside a client deliverable. Layers run from L0 identity through L4 working artifacts, with measurable interpretability and reproducibility gains reported by a 52-member practitioner community.

When a regulated client asks how a model reached a conclusion, an interpretable folder layer is a better answer than a black box. That is the part EisnerAmper's clients, and EisnerAmper's own audit partners, will care about.
